Cases Page Help

Cases Tab

The Cases tab contains a list of research and clinical cases that may demonstrate phenotypes due to disruption of TADs or other non-coding elements. Cases can be sorted by clicking on the arrow for each column. Cases can be filtered by clicking on the title of each column (optional), entering text to search for in the Filter form, and then pressing enter.

Cases have the following properties
  • Status: cases are "Pending" after submitted by users and "Confirmed" after review by a moderator
  • Evidence: the evidence supporting pathogenicity
  • Submitter: the user who submitted the case to ClinTAD
  • Build: the genome build for the case's coordinates
  • Coordinates: the chromosome and breakpoints for the CNV
  • Phenotypes: a list of HPO terms for the patient/subject associated with the case
  • Comments: submitters can enter other important details here

The Create Case tab can be used to submit cases where patients/subjects are thought to have phenotypes due to disruption of non-coding elements. After submission, these cases will be publicly viewable and should not contain Protected Health Information.

Cases should have the following information, if available:
  • Build: the genome build for the coordinates
  • Coordinates: the chromosome and breakpoints, in UCSC browser format e.g. chr1:5000000-6000000.
  • PubMed IDs: the PubMed IDs for any publications that describe the case, if entering multiple separate with commas.
  • HPO Phenotypes: the HPO identifiers for the phenotypes associated with the patient/subject. To lookup IDs, enter a search term in the Lookup HPO Term box and press enter. Then select a term from the HPO list and press the Add button. Multiple values should be separated by commas.
  • Evidence for Case: select any evidence for the case supporting the relationship between patient phenotypes and disruption of non-coding elements.
  • Comments: enter any other information that is relevant to the case.
